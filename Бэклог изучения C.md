# Бэклог изучения C

Цель бэклога - превратить большой план и упражнения в понятную очередь задач. Отмечай выполненное чекбоксами и не переходи к следующему крупному блоку, пока не закрыт критерий готовности текущего блока.

Режим обучения: интенсивный. Если есть много времени, нормальный темп - 2-4 часа в день: теория, код, разбор ошибок, короткий конспект.

## Как пользоваться

- `P0` - обязательно, без этого дальше будет тяжело.
- `P1` - важно, закрепляет тему.
- `P2` - полезно, но можно отложить.
- `S` - маленькая задача, до 30 минут.
- `M` - средняя задача, 30-90 минут.
- `L` - большая задача, 2+ часа.

Правило закрытия задачи:

- код компилируется без предупреждений;
- программа проверена на нескольких входных данных;
- ошибки записаны в Obsidian;
- ты можешь вслух объяснить, что делает каждая строка.

Команда проверки:

```bash
gcc -Wall -Wextra -Wpedantic -std=c11 main.c -o main
```

Пока ты ноль или почти ноль в C, главный фокус:

Zynq, TCP/IP, DMA, HLS и ПЛИС пока держи как конечную цель, но не как ежедневную основную тему.

# Статус по текущему коду

Отмечено по файлам `block1.c` и `block2.c`.

Сделано:

- `block1.c`: калькулятор, максимум из трех чисел, таблица умножения, сумма цифр, переворот числа.
- `block2.c`: ввод динамического массива, `malloc`, проверка результата `malloc`, ввод элементов через указатель, статистика массива частично, поиск элемента, разворот массива, освобождение памяти через `free`.

Пока не зачтено как готовое:

- проверка високосного года: правило неполное;
- среднее значение массива: сейчас целочисленное деление;
- подсчет нулей: нули сейчас попадают в `Positive`;
- `sort_array`: есть утечка памяти из-за потери указателя после `malloc`;
- проверка `scanf` и проверка размера массива: пока отсутствуют.
