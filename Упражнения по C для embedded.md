## Правила выполнения

- Каждая задача должна компилироваться без предупреждений:

```bash
gcc -Wall -Wextra -Wpedantic -std=c11 main.c -o main
```

- Для каждой задачи пиши отдельную функцию, если это не совсем маленький пример.
- Не оставляй всю логику в `main`.
- После каждой задачи записывай в Obsidian:
  - что программа делает;
  - какие ошибки были;
  - где похожая логика встречается в микроконтроллерах, Zynq или ПЛИС-проектах.
- Если задача связана с массивом, всегда проверяй размер массива.
- Если задача связана с делением, всегда проверяй деление на ноль.

# Блок 1. База C

Цель: уверенно писать простые программы с переменными, условиями, циклами и функциями.

## 1. Калькулятор

Статус: сделано в `block1.c`.

- Написать функцию `int calc(int a, int b, char op)`.
- Поддержать операции `+`, `-`, `*`, `/`, `%`.
- При делении на ноль вывести ошибку.
- При неизвестной операции вывести ошибку.

Готово, если:

- `calc(10, 5, '+')` возвращает `15`;
- `calc(10, 0, '/')` не ломает программу;
- `calc(10, 5, '?')` сообщает об ошибке.

## 2. Максимум и минимум

Статус: сделано в `block1.c`.

- Написать `int max_of_three(int a, int b, int c)`.
- Написать `int min_of_three(int a, int b, int c)`.
- Проверить на положительных, отрицательных и одинаковых числах.

## 3. Четность

Статус: сделано в `block1.c`.

- Написать `int is_even(int x)`.
- Функция должна возвращать `1`, если число четное, и `0`, если нечетное.
- Проверить `0`, положительные и отрицательные числа.

## 4. Високосный год

Статус: проверка сделана в `block1.c` через `verify_year`, но отдельную функцию `is_leap_year` еще стоит добавить.

- Написать `int is_leap_year(int year)`.
- Правило:
  - год делится на 400: високосный;
  - год делится на 100: не високосный;
  - год делится на 4: високосный;
  - иначе не високосный.

Проверить годы: `2000`, `1900`, `2024`, `2023`.

## 5. Таблица умножения

Статус: сделано в `block1.c`.

- Вывести таблицу умножения от `1` до `10`.
- Сделать вывод аккуратным, чтобы строки не слипались.

## 6. Сумма цифр числа

- Написать `int sum_digits(int number)`.
- Для `5382` результат должен быть `18`.

## 7. Переворот числа

Статус: сделано в `block1.c`, функция уже называется `reverse_number`.

- Написать `int reverse_number(int number)`.
- Для `1234` результат должен быть `4321`.
- Проверить число с нулем в конце: `1200`. Для числового результата нормально получить `21`, потому что `0021` в `int` не хранится.
- Дополнительно: обработать отрицательное число.

Цель: научиться работать с наборами данных и не выходить за границы массива.

## 2. Статистика массива

Статус: почти сделано в `block2.c`. Сумма, минимум, максимум, среднее, положительные и отрицательные считаются; нули пока надо вынести в отдельный счетчик.

Написать функцию:

```c

```

Она должна вывести:

- сумму;
- минимум;
- максимум;
- среднее значение;
- количество положительных;
- количество отрицательных;
- количество нулей.

Готово, если среднее для массива `{1, 2, 2}` выводится как `1.666...`, а не как `1.000...`.

## 3. Поиск элемента

Статус: сделано в `block2.c`, но текущая функция сама спрашивает число у пользователя. Позже лучше передавать `value` параметром. ПОнял будем считать что сделал!

Написать:

```c

```

Функция возвращает индекс первого найденного элемента или `-1`.

## 4. Разворот массива

Статус: сделано в `block2.c`.

Написать:

```c

```

Проверить массивы:

- `{1, 2, 3, 4}`;
- `{1, 2, 3, 4, 5}`;
- `{1}`;
- пустой массив с размером `0`.

## 5. Сортировка

Статус: сделано в `block2.c`. Функция возвращает новую отсортированную копию массива, исходный массив не меняется.

Написать:

```c

```

Можно начать с пузырьковой сортировки.

Важно:

- если функция сортирует исходный массив, ей не нужен `malloc`;
- если нужен новый массив, надо явно скопировать данные;
- нельзя выделить память через `malloc`, а потом потерять этот адрес.

Твой текущий вариант выбрал второй путь: `sort_array` выделяет новый массив, копирует туда исходный и сортирует копию.
