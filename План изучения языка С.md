# Этап 8. `volatile`, `const` и регистры

## Цель этапа

Понять специальные ключевые слова, которые особенно важны в embedded.

## `volatile`

`volatile` говорит компилятору:

> Не оптимизируй эту переменную, её значение может измениться вне программы.

Это нужно для:

- регистров;
- флагов прерываний;
- DMA;
- переменных, изменяемых в ISR.

Пример:

```c
volatile uint32_t *reg = (uint32_t*)0x40000000;
```

## `const`

`const` запрещает менять значение.

```c
const int x = 10;
```

## Важные варианты

```c
const int *p;        // указатель на константные данные
int *const p;        // константный указатель
const int *const p;  // константный указатель на константные данные
```

## Практика

- разобрать `const int *p`;
- разобрать `int *const p`;
- разобрать `volatile uint32_t *reg`;
- написать пример работы с воображаемым регистром.

## Варианты программирования

### Bare-metal

Код работает без операционной системы.

Плюсы:

- предсказуемость;
- меньше накладных расходов;
- проще работать с регистрами.

Минусы:

- сложнее сеть;
- сложнее файловые системы;
- сложнее многозадачность.

### Linux

Код работает как обычная программа в Linux.

Плюсы:

- TCP/IP уже есть;
- можно использовать sockets;
- проще серверы;
- проще файловая система;
- проще отладка.

Минусы:

- меньше real-time;
- нужно учитывать ОС;
- доступ к железу сложнее.

## Что особенно важно для твоих задач

Для проекта с Zynq, Ethernet, DDR, TCP/IP и DAC нужно понимать:

- C на уровне Linux sockets;
- работу с буферами;

- `uint8_t`, `uint32_t`;
- DMA;
- cache flush/invalidate;

# Критерий готовности к микроконтроллерам

Можно переходить к микроконтроллерам, когда ты уверенно понимаешь такой код:

```c
#define LED_PIN 5
#define BIT(x) (1U << (x))

volatile uint32_t *GPIO_ODR = (uint32_t*)0x40020014;

void led_on(void)
{
    *GPIO_ODR |= BIT(LED_PIN);
}

void led_off(void)
{
    *GPIO_ODR &= ~BIT(LED_PIN);
}

int main(void)
{
    while (1)
    {
        led_on();
        led_off();
    }

    return 0;
}
```

Если понятно, что здесь происходит, значит база для embedded C уже появилась.
